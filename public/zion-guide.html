Annotator Reference Guide

Zion Annotation Guidelines

A semantic annotation pipeline for auditing and refining AI-generated captions for short-form video content. Experts review seed captions, verify against video, and edit or approve.

Pillar 01

Correctness

Each annotation must accurately reflect the visual data — no hallucinations, no inferred details.

Pillar 02

Completeness

All relevant hand actions and segments are captured, with no gaps in the sequence.

Pillar 03

Consistency

Object descriptions and action labels remain uniform across the entire annotation.

The Central Question Zion Evaluates

Is this annotation reliable enough to be used as training data?

Handling Time

1:5 ratio — 5 seconds of work per 1 second of video

Zion annotations set a benchmark for model performance. Consistent Handling Time (HT) ensures the model learns from realistic, repeatable task execution.

2.1

Caption Validation

Seven pillars — each defines what to check and how to fix it
Pillar 01 · Accuracy

Action Verification

Check

Did the described actions actually occur?

Fix

Remove any descriptions of actions not visible in the video. If a major action is missing, add a concise description.

Pillar 02 · Identity

Object & Attribute Precision

Check

Are all entities identified correctly?

Fix

Verify the color, type, and shape of objects. If the model identifies a red truck but the video shows a maroon van, update immediately.

Pillar 03 · Order

Chronological Sequencing

Check

Does the caption follow the video's timeline?

Fix

Ensure the narrative follows the visual sequence. If events are described out of order, restructure to match chronology.

Pillar 04 · Flow

Annotation Logic & Action Flow

Check

Is there a beginning, middle, and end for the object?

Fix

Maintain a coherent action sequence: Pick → Use → Place. Avoid broken continuity. Consolidate fragmented actions to preserve flow.

Pillar 05

Pauses

Check

Should indecision, hesitation, or frustration be annotated?

Fix

No. Only annotate observable, task-relevant actions.

Pillar 06

No-Action Segments

Check

How should periods with no actions be handled?

Fix

Label as "No relevant actions."

Pillar 07

Independent Object Descriptions

Check

Should descriptions rely on previous steps to identify an object?

Fix

No. Use standalone descriptions: e.g. "a blue plastic cup" — not "the cup currently placed on the table."

Decision Framework

Yes = pass · No = apply the fix
1
Did all actions described happen in the video?
✓ Yes — Pass✗ No — Remove actions that did not occur
2
Did all actions occur in the right order?
✓ Yes — Pass✗ No — Switch to correct order
3
Are all the colors / objects described correctly?
✓ Yes — Pass✗ No — Change the description
💡 Rule of Thumb
If it didn't happen → delete it
If it's the wrong color / type → change it
If the timing is off → move it

Captioning Conventions

Six rules governing how captions are written
01

Movement

NO

Hands only — no body movements. Avoid "walks", "steps", "moves the legs", "torso".

✕ "moves four steps forward and picks up a box"
✓ "moves forward, reaches for, and picks up a box"
02

Viewpoint Changes

NO

Only label left and right hand actions. Exclude head movement and camera perspective changes.

03

Object Identification

Reference only

Use objects as reference points for hand actions — not as standalone mentions.

04

Action Outcomes

YES

If an action results in a secondary event, include it.

✓ "The right hand places a cup on the counter and a finger knocks it over."
05

Object Generalization

YES

If the material is hard to identify, use a general term rather than guessing.

✓ Use "cup" instead of "plastic / glass cup" when material isn't clear.
06

Hand Rest Positioning

YES

Always describe the hand's resting position at end of each segment. High default = above waist. Low default = below waist.

✓ "The right hand places the cup, then returns to a high default position."

Error Categories

Tag every category that applies — a single fix often needs multiple tags
action_fix

Action Fix

Wrong verb — doesn't match what's happening.

e.g. "shakes the bottle" but hand actually rotates it.
timestamp_fix

Timestamp Fix

Caption correct but window timing is off.

e.g. Window 0:03→0:09 when action ends at 0:07.
hand_fix

Hand Fix

Wrong hand label — left/right swapped, or active/idle mislabeled.

e.g. "Left hand reaches" when it's the right.
object_fix

Object Fix

Wrong object label, color, or descriptor.

e.g. "maroon cup" instead of "red cup".
hallucination

Hallucination

Invented object, action, or detail not in the video.

e.g. Caption mentions a brand not visible in the video.
missing_phase

Missing Phase

Gap in the timeline where an action occurs but isn't annotated.

e.g. A reach between two captions has no phase of its own.
merge_segments

Merge Segments

One continuous action split into multiple short captions.

e.g. A single reach broken into three 1-second captions.
split_segments

Split Segments

One caption covers more time than the action it describes.

e.g. "Picking up cup" 0:03→0:07 but pickup ends at 0:05.
structural_cleanup

Structural Cleanup

Phrasing off but facts correct.

e.g. Awkward word order; subject at end of sentence.

Ex 1 · Unverifiable object brand
Hallucination — invented brand name
✕ Incorrect

Both hands carry an upside-down Olavson copper pan, gripping opposite edges of the pan to transport it steadily.

✓ Correct

Both hands carry an upside-down copper pan, gripping opposite edges.

Error: "Olavson" is an invented brand name not verifiable from the video. Describe only what is visibly verifiable.
Ex 2 · Hallucinated product type
Hallucination — guessed product flavor
✕ Incorrect

The right hand sets a green lime soda can into a wooden counter while the left hand remains idle.

✓ Correct

The right hand sets a green can onto a wooden counter while the left hand remains idle.

Error: Unable to verify if the can is Lime Soda. Use the visible attribute ("green can") only.

Ex 1 · Left hand incorrectly labeled as active
hand_fix — left hand is idle, not active
✕ Incorrect

The right hand places a white cup in the sink, while the left hand reaches for the pan handle but fails to grasp it.

✓ Correct

The right hand places a white cup in the sink, while the left hand remains still.

Error: Left hand is idle.
Ex 2 · Left hand incorrectly labeled as grabbing
hand_fix — left hand is idle
✕ Incorrect

The left hand grabs the wooden counter while the right hand lifts a small metal pot over the sink basin.

✓ Correct

The left hand remains idle while the right hand lifts a small metal pot over the sink basin.

Error: Left hand is idle.
Ex 3 · Wrong hand — left, not right
hand_fix — hands swapped
✕ Incorrect

The right hand reaches out to grasp and lift a blue-capped water bottle from the counter while the left hand remains resting on the edge.

✓ Correct

The left hand rests open on the right side of the table, then lifts, moves backward to the left, and reaches for the blue-capped water bottle to grasp it, while the right hand remains idle.

Error: The left hand grabs the bottle; the right hand is idle.
Ex 4 · Wrong hand — left hand is active, not right
hand_fix — active hand swapped
✕ Incorrect

The right hand reaches into a white bowl, briefly touches a green snack bar, and then picks up and removes the blue snack bar while the left hand remains out of view.

✓ Correct

The left hand reaches into a white bowl and picks up a blue snack bar, while the right hand remains out of view.

Error: The left hand is active; the right is off-screen.
Ex 5 · Wrong hand — right hand receives, not left
hand_fix — receiving hand swapped
✕ Incorrect

The left hand reaches out and takes a green bottle offered by a person, while the right hand remains out of view.

✓ Correct

The right hand reaches out and takes the green bottle, while the left hand remains out of view.

Error: The robot reaches with its right hand, not the left.
action_fix Examples
Ex 1 · Wrong verb — pick up & rotate, not shake
action_fix — shake vs. rotate
✕ Incorrect

The right hand shakes the silver travel mug from the desk while the left hand remains idle.

✓ Correct

The right hand picks up the silver travel mug from the desk and rotates it horizontally, while the left hand remains idle.

Error: The correct action is pick up and rotate, not shake.
Ex 2 · Wrong verb — reach for, not grasp
action_fix — grasp vs. reach for
✕ Incorrect

The left hand grasps and lifts a water bottle from the table, rotating it horizontally, while the right hand remains stationary holding a clear plastic cup.

✓ Correct

The left hand reaches for and lifts a water bottle from the table, rotating it horizontally, while the right hand remains stationary, holding a clear plastic cup.

Error: Since the hand must travel a significant distance, "reaches for" is more accurate than "grasps."
Ex 3 · Wrong verb — grab & lift, not rotate
action_fix — rotate vs. grab and lift
✕ Incorrect

The right hand rotates a clear, blue-capped water bottle on a counter while the left hand hovers nearby without interacting.

✓ Correct

The right hand grabs and lifts a clear, blue-capped water bottle from the counter while the left hand remains idle.

Error: The hand never rotates the bottle — the correct action is grabbing and lifting.
missing_phase Examples
Ex 1 · Multi-step reach collapsed into "pick up"
missing_phase — collapsed sequence
✕ Incorrect

pick up Fiji water bottle with right hand

✓ Correct

The right hand rests below the table. The right hand reaches for the bottle and picks it up, while the left hand remains idle.

Error: The right hand performs multiple actions before picking up the bottle; the left hand is visible and idle — not absent.
Ex 2 · Five-step pickup reduced to a single reach
missing_phase — five distinct motions collapsed
✕ Incorrect

The right hand reaches forward and grasps a blue bottle on a wooden table while the left hand remains idle.

✓ Correct

The right hand rests closed on the table to the right of the water bottle, then moves backward, opens, and moves forward to pick up the bottle while the left hand is out of frame.

Error: The hand performs five distinct actions, and the left hand is out of frame — not idle.
object_fix Examples
Ex 1 · Wrong object — lime, not ball
object_fix — green ball vs. lime
✕ Incorrect

The right hand grasps and lifts a green ball from the container while the left hand remains idle.

✓ Correct

The right hand grasps and lifts a lime from the container while the left hand remains idle.

Error: The video clearly shows a lime, not a green ball. Fix obvious object misidentifications.
Ex 2 · Unverifiable fruit — generalize to "object"
object_fix — unverifiable category
✕ Incorrect

The left hand holds a Rubik's cube while the right hand places a yellow-orange fruit into a cardboard box and then picks up a yellow lemon.

✓ Correct

The left hand holds a Rubik's cube while the right hand grabs a yellow-orange object from a cardboard box.

Error: The object can't be confidently identified as a fruit — use "object" rather than guessing a category.
Ex 3 · Wrong scope — whole bottle, not just the cap
object_fix — wrong object scope
✕ Incorrect

The right hand reaches for and picks up a blue cap from the counter while the left hand remains idle.

✓ Correct

The right hand moves backward, grasps a blue-capped water bottle, and lifts it from the counter while the left hand remains idle.

Error: The hand picks up the whole bottle, not just the cap.

Anatomy of a Task

A task = one video broken into multiple captioned time windows

Each task is a single video pre-segmented into multiple captions. Each caption is the atomic reviewable unit: a numbered [start, end] time window, paired with a hand-action description and object list.

FieldTypeDescription
IDintegerSequential number identifying the caption within the video.
TrackscopeWhich hand(s) the annotation covers — left, right, or both.
Time Window[start, end]The bounded video slice defining the caption's footprint on the timeline.
DescriptiontextNatural-language caption of the hand actions. Must account for both hands, even if one is idle or out of frame.
Objectstag listObjects relevant to the caption, as standalone descriptors used as reference points for hand actions.

End-to-End Workflow

Three stages: claim → review → submit
1

Claim a task

Pick up an available task from the queue. Each task is a single video. Once claimed, complete it in full.

2

Review caption by caption

For each caption, run the same micro-loop before advancing:

a. Read the model-generated description carefully.

b. Watch the clip — confirm both hands and all objects match.

c. Confirm start and end times tightly bound the action.

d. Apply corrections, then move to the next caption.

3

Save & submit

Once every caption has been reviewed, save and submit. Partial reviews aren't submitted.

⏱ RememberThe HT budget covers the entire workflow — claiming, full review loop, and submission. Keep the per-caption loop tight to stay within the 1:5 ratio.

Annotation Refinement Examples

Seed captions vs. expert-refined versions
Example 1 · Caption #0 · 0:00→0:05 · Adding physical locomotion
Refinement example — squeegee grab
✕ Original

The right hand pulls the green and white squeegee from the wall hook; the left hand takes the squeegee from the right hand and lowers the blade onto the spilled yellow liquid on the white counter while the right hand becomes idle.

✓ Refined

The right hand reaches for the green and white squeegee and grabs it from the wall hook behind the sink, then both hands move to the right, and the left hand takes the squeegee from the right hand and lowers the blade onto the top-right corner of the counter, while the right hand becomes idle.

Changes made

Added physical locomotion — the hand movement to grab the squeegee and move to the right must be included to complete the sequence.

"reaches for" · "both hands move to the right"
Example 2 · Caption #4 · 0:18→0:25 · Specifying exact placement destinations
Refinement example — scissors and paintbrush
✕ Original

The right hand picks up a black scissors and wooden paintbrush from the desk and places them into the clear plastic tray of the blue and clear toolbox while the left hand stays idle.

✓ Refined

The right hand picks up the black scissors from the desk and places them on top of the paint pots inside the blue compartment, then picks up the wooden paintbrush and places it into the transparent compartment on the upper section of the toolbox, while the left hand remains idle throughout.

Changes made

Added precise placement destinations for each object and separated the two placement actions into sequential steps.

"on top of the paint pots inside the blue compartment" · "into the transparent compartment on the upper section"
Example 3 · Caption #7 · 0:00→0:10 · Correcting hand state and final action accuracy
Refinement example — whiteboard eraser
✕ Original

The right hand scrubs the whiteboard with the pink eraser, lifts it away from the surface, and places it on the top frame of the whiteboard while the left hand stays idle.

✓ Refined

The right hand scrubs the whiteboard with the pink eraser while the left hand rests on the upper frame. The right hand releases the eraser onto the whiteboard and moves toward the upper frame as the left hand lifts away, leaving both hands suspended at that height without touching it.

Changes made

Two corrections: (1) the left hand was not idle — it rested on the upper frame. (2) The eraser is released back onto the whiteboard surface magnetically, not placed on the frame.

"left hand rests on the upper frame" · "releases the eraser onto the whiteboard" · "both hands suspended without touching it"

Common Errors

Patterns spotted in recent reviews

Inconsistent Action Granularity

missing_phasestructural_cleanup

Every distinct hand motion gets its own verb — not a compressed summary.

grasp
lift
approach
place
✕ Too high-level

The right hand picks up and throws an item in A.

✓ Atomic verbs

The right hand grasps the item, lifts it from the surface, approaches A, and places it inside.

Why it matters: Each atomic verb corresponds to a discrete motion the model can reproduce. Compressed verbs confuse training.

Positional Numbering of Objects

object_fix

Never label objects by sequence position. Always describe by visible physical attributes: color, shape, size, location.

✕ Avoid

The right hand picks up the second cup from the counter.

The left hand places the first bottle into the box.

The right hand grasps the third snack bar from the bowl.

✓ Use instead

The right hand picks up the red cup from the counter.

The left hand places the tall blue bottle into the box.

The right hand grasps the green snack bar from the bowl.

Why it matters: "Second cup" tells the model nothing about which object to grasp. "Red cup" grounds the action in a visual attribute the model can learn from.
